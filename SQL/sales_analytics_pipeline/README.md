# Sales Analytics Pipeline - SQL Analysis

SQL practice using joins, CTEs, and a window function (`LAG`) to answer basic sales questions. The database (`database/sales_warehouse.db`) is built by the Python part and already included here, no setup needed.

Other parts: [Python](../../Python/sales_analytics_pipeline) · [Power BI](../../PowerBI/sales_analytics_pipeline)

## Dataset

Fake sales data (customers, products, orders), generated by the Python script.

## Tools

- SQLite
- DB Browser for SQLite
- Git / GitHub

## Queries (`analytics_queries.sql`)

1. Join orders, customers, and products.
2. Monthly revenue and month-over-month growth (CTE + `LAG`).
3. Top 5 customers by revenue (CTE).
4. Product performance by category (CTE).
5. Extra: revenue by customer segment.
6. Extra: order status breakdown.

## Findings

- ~$7.28M total revenue from 5,133 completed orders.
- Revenue was flat between 2023 ($3.63M) and 2024 ($3.65M).
- Software is the top category by revenue.
- The three customer segments are almost tied in revenue.
- ~12-13% of orders end up Cancelled or Returned.

## How to run

```bash
cd SQL/sales_analytics_pipeline
sqlite3 database/sales_warehouse.db < analytics_queries.sql
```
